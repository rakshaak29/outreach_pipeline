# Automated Outreach Pipeline

A production-ready Python pipeline that, given a seed company domain:

1. **Finds similar companies** via the [Ocean.io](https://ocean.io) API (`POST /v3/search/companies`).
2. **Discovers decision-makers** (CEO, CTO, Founder, VP Engineering, VP Sales, etc.) via the [Prospeo](https://prospeo.io) API.
3. **Generates personalised plain-text outreach messages** per contact.
4. **Stores all results** in a structured JSON file.
5. **Exposes an interface** (`OutreachInterface`) where email sending can be plugged in later.

## Configuration

Copy `.env` and set at minimum:

```dotenv
OCEAN_API_KEY=your_ocean_api_token_here
PROSPEO_API_KEY=your_prospeo_api_key_here
```

All other variables have sensible defaults (see `.env` for descriptions).

| Variable | Default | Description |
|---|---|---|
| `OCEAN_API_KEY` | *(required)* | Ocean.io API token (Account Settings → API Tokens) |
| `PROSPEO_API_KEY` | *(required)* | Prospeo API key |
| `OCEAN_BASE_URL` | `https://api.ocean.io/v3` | Ocean.io base URL (lookalike endpoint: `{url}/search/companies`) |
| `PROSPEO_BASE_URL` | `https://api.prospeo.io` | Prospeo base URL |
| `MAX_SIMILAR_COMPANIES` | `10` | Companies to fetch per run |
| `MAX_CONTACTS_PER_COMPANY` | `5` | Contacts per company |
| `RETRY_MAX_ATTEMPTS` | `3` | Total HTTP attempts before aborting |
| `RETRY_BACKOFF_FACTOR` | `1.5` | Backoff multiplier in seconds |
| `OUTPUT_FILE` | `data/results.json` | Results file path |
| `LOG_LEVEL` | `INFO` | Python logging level (`DEBUG` shows full request/response) |

## Usage

### Basic run

```bash
python main.py --domain notion.so
```

### Dry-run mode (no emails sent — messages generated and displayed only)

```bash
python main.py --domain notion.so --dry-run
```

### Ocean-only mode (skip Prospeo contact lookup entirely)

```bash
python main.py --domain notion.so --ocean-only
```

Useful when Prospeo is rate-limited or you only need the company list.
All companies will have `contacts_fetch_status: "skipped"`.

### Resume mode (re-attempt companies that were rate-limited or failed)

```bash
python main.py --domain notion.so --resume
```

Loads the previous `results.json`, keeps companies with
`contacts_fetch_status: "success"` as-is, and re-attempts the rest.

### Limit results

```bash
python main.py --domain notion.so --max-companies 5 --max-contacts 3
```

### Export to CSV (from accumulated SQLite data)

```bash
python main.py --export-csv
```

Exports all data from every previous run into:
- `exports/companies.csv` — all company rows with `run_id`, `seed_domain`, `contacts_fetch_status`, etc.
- `exports/contacts.csv`  — all contact rows with `run_id`, `company_domain`, email, title, outreach message, etc.

Does **not** run the pipeline — purely a read from SQLite.

### Override the database file

```bash
python main.py --domain notion.so --db /path/to/custom.db
python main.py --export-csv --db /path/to/custom.db
```

### Full example

```bash
python main.py \
  --domain notion.so \
  --dry-run \
  --max-companies 10 \
  --max-contacts 5 \
  --output data/notion_results.json
```

## Diagnostics

If Ocean.io returns 404, run the standalone diagnostic to probe all known endpoints
and confirm your API key is valid:

```bash
python test_ocean.py --verbose
```

The script:
- Loads `OCEAN_API_KEY` from `.env`
- Probes every known endpoint path across v2 and v3
- Prints full request/response details (key masked)
- Confirms whether your key is valid and your plan has access
- Reports `creditsUsed` per call and your remaining balance

Enable `DEBUG` logging in the pipeline itself by setting `LOG_LEVEL=DEBUG` in `.env`.
This prints the full request URL, headers (masked), body, status code and response body
for every Ocean.io and Prospeo call.

## SQLite Persistence

Every pipeline run automatically writes to `database.db` (SQLite, auto-created).

### Tables

| Table | Description |
|---|---|
| `pipeline_runs` | One row per run: `run_id`, `source_domain`, `timestamp`, `dry_run`, `ocean_only`, `companies_found`, `contacts_found` |
| `companies` | One row per company per run: all fields from the Ocean.io response plus `contacts_fetch_status` |
| `contacts` | One row per contact per run: all Prospeo enrichment fields plus `outreach_message` |

All tables carry `run_id` as a foreign key, so you can JOIN across runs or query a single run's results.

### Quick DB inspection

```bash
# Open with the sqlite3 CLI
sqlite3 database.db

sqlite> .tables
sqlite> SELECT source_domain, companies_found, contacts_found FROM pipeline_runs;
sqlite> SELECT domain, name, contacts_fetch_status FROM companies;
sqlite> SELECT full_name, email, title, company_domain FROM contacts;
```

### CSV Export

```bash
python main.py --export-csv
```

Reads **all** rows from SQLite (across every run) and writes:

| File | Columns |
|---|---|
| `exports/companies.csv` | `id`, `run_id`, `domain`, `name`, `industry`, `employee_count`, `location`, `website`, `description`, `similarity_score`, `contacts_fetch_status`, `fetched_at`, `seed_domain`, `run_timestamp` |
| `exports/contacts.csv` | `id`, `run_id`, `company_domain`, `full_name`, `first_name`, `last_name`, `email`, `title`, `linkedin_url`, `seniority`, `department`, `person_id`, `outreach_message`, `fetched_at`, `seed_domain`, `run_timestamp` |

> **Note**: `database.db` and `exports/` are both git-ignored by default.

  "contacts_fetch_summary": {
    "success": 5,

  "companies": [
    {
      "domain": "coda.io",
      "name": "Coda",
      "industry": "SaaS",

      "location": "United States",

    }

### `contacts_fetch_status` values

| Value | Meaning |
|---|---|
| `success` | Prospeo returned (contacts may be an empty list if none matched) |
| `rate_limited` | Prospeo returned HTTP 429; retry with `--resume` |
| `failed` | Prospeo returned another API error; retry with `--resume` |
| `skipped` | Prospeo was not called (`--ocean-only` mode) |

## Error Handling
## Rate Limiting & Resilience

The pipeline is designed to **never crash** due to Prospeo rate limits:

| Scenario | Behaviour |
|---|---|
| Prospeo returns 429 on company N | Company N marked `rate_limited`; remaining companies also marked `rate_limited` without further calls |
| Prospeo returns another 4xx | Company marked `failed`; pipeline continues to next company |
| `--ocean-only` flag used | All companies marked `skipped`; Prospeo not called at all |
| Pipeline interrupted mid-run | Partial results already written to `results.json` |
| `--resume` flag used | Previous successes kept; rate_limited / failed companies retried |

**Typical recovery workflow when rate-limited:**

```bash
# First run — Ocean.io works, Prospeo hits rate limit at company 2
python main.py --domain notion.so --max-companies 10

# Wait for Prospeo rate limit to reset, then:
python main.py --domain notion.so --resume
# → Companies 1 (already success) kept; companies 2-10 retried
```

---

## Error Handling

| Scenario | Behaviour |
|---|---|
| Missing API key | `EnvironmentError` at startup — pipeline aborts with clear message |
| Invalid domain | `ValidationError` — pipeline aborts with usage hint |
| 401 / 403 from Ocean.io | `AuthenticationError` — pipeline aborts |
| 401 / 403 from Prospeo | `AuthenticationError` — pipeline aborts |
| 429 from Prospeo | `RateLimitError` caught per-company — marked `rate_limited`, pipeline continues |
| 5xx server errors | Retried with exponential backoff; company marked `failed` after exhaustion |
| Empty API response | `WARNING` logged; company / contact skipped silently |
| File write failure | `ERROR` logged; results printed to stdout as fallback |

## Plugging In Email Sending (Future)

To add email delivery, implement the `OutreachInterface` ABC in `services/outreach_service.py`:

```python
from services.outreach_service import OutreachInterface, OutreachService
from models.contact import Contact

class SendGridAdapter(OutreachInterface):
    def send(self, message: str, contact: Contact) -> None:
        # your SendGrid / SMTP logic here
        sendgrid_client.send(to=contact.email, body=message)

# Pass your adapter to OutreachService
outreach = OutreachService(adapter=SendGridAdapter(), dry_run=False)
```

No other files need to change.

## Architecture Notes

- **`requests.Session`** is used throughout for connection pooling and consistent headers.
- **`@retry` decorator** in `utils/retry.py` handles transient failures centrally — service code stays clean.
- **`Config` is a frozen dataclass** loaded once and cached — thread-safe by design.
- **`OutreachService`** is decoupled from delivery — the `OutreachInterface` ABC is the only seam needed to add email sending.
- **Pydantic v2 models** provide runtime validation, type coercion, and `.model_dump_json()` serialisation with zero boilerplate.
